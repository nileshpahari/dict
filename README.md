# dict
(README is generated by an LLM)

A fast, minimal personal dictionary for the command line. Store and look up word meanings locally, with no internet required.

Words are persisted to a plain-text file (`dict.txt`) and loaded into an in-memory hash map on startup.

---

## Build

```sh
gcc -Wall -Wextra -o dict main.c cli.c hashmap.c storage.c
```

---

## Usage

`dict` runs in two modes: **single-command** and **interactive**.

### Single-command mode

```sh
dict <command> [arguments]
```

| Command | Description |
|---|---|
| `dict add <word> <meaning>` | Add or update a word |
| `dict search <word>` | Look up a word |
| `dict delete <word>` | Remove a word |
| `dict help` | Show usage |

**Examples**

```sh
$ dict add ephemeral lasting for a very short time

Word added successfully.

$ dict search ephemeral

  Word: ephemeral

  Meaning: lasting for a very short time

$ dict delete ephemeral

Word deleted successfully.
```

Multi-word meanings work naturally — no quoting needed:

```sh
$ dict add serendipity the occurrence of events by chance in a happy way
```

### Interactive mode

Launch with no arguments to enter a REPL session:

```sh
$ dict

Usage: <command> <argument>
Type "help" for listing possible commands or "exit" for exiting interactive mode.

>>> add ubiquitous present, appearing, or found everywhere
Word added successfully.
>>> search ubiquitous

  Word: ubiquitous

  Meaning: present, appearing, or found everywhere

>>> delete ubiquitous
Word deleted successfully.
>>> exit
```

## Project structure

```
dict/
├── main.c        Entry point. Loads data, dispatches to CLI, saves on exit.
├── cli.c/h       Command parsing for both single-command and interactive modes.
├── hashmap.c/h   Hash map implementation (djb2, chaining, auto-resize at 0.75 load).
├── storage.c/h   File I/O — loads and saves dict.txt using a safe tmp-then-rename strategy.
└── dict.txt      Persistent store (tab-separated: word<TAB>meaning, one per line).
```

### How it works

1. On startup, `dict.txt` is read line-by-line into a hash map (capacity 100, resizes as needed).
2. The requested command runs against the in-memory map.
3. On exit, the full map is written atomically to `dict.txt` via a temporary file + `rename(2)`.

The hash map uses **djb2** hashing with **separate chaining** for collision resolution. It doubles in capacity whenever the load factor exceeds 0.75.
